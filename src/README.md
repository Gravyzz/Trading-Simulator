# Trading Simulator

Консольный симулятор биржевой торговли на чистой Java. Позволяет покупать и продавать акции, отслеживать портфель с расчётом прибыли, анализировать вложения и совершать рыночные и лимитные ордера. Ключевая особенность — цены акций изменяются в реальном времени в фоновом потоке, имитируя живой рынок.

Проект написан без сторонних библиотек и фреймворков — только стандартная Java. Цель — продемонстрировать владение всеми основными разделами Java Core: ООП и наследованием, обобщёнными коллекциями, Stream API, обработкой исключений, многопоточностью и работой с файлами.

---

## Возможности

- Просмотр рынка с текущими ценами акций по секторам
- Покупка и продажа акций с автоматическим расчётом средней цены позиции
- Портфель с учётом баланса, позиций и прибыли/убытка по каждой бумаге
- Рыночные и лимитные ордера (лимитный исполняется только при достижении заданной цены)
- Аналитика портфеля: общая стоимость, суммарная прибыль, самая ценная позиция, ранжирование по стоимости, список прибыльных бумаг
- Цены обновляются в фоновом потоке в реальном времени, пока пользователь торгует
- Сохранение истории сделок в файл и загрузка при следующем запуске
- Обработка ошибок ввода: неверный тикер, нехватка средств, некорректное количество

## Технологии и концепции

| Область | Что использовано |
|---|---|
| ООП | Инкапсуляция, иммутабельные объекты, `equals` / `hashCode`, композиция |
| Наследование и полиморфизм | Абстрактный класс `Order`, подклассы `MarketOrder` и `LimitOrder`, переопределение методов |
| Enum | `Sector` и `TradeType` с полями и методами |
| Коллекции | `Map` (тикер → акция, тикер → позиция), `List`, защитное копирование |
| Stream API | `filter`, `map`, `mapToDouble`, `sum`, `sorted`, `max`, `Comparator`, `Optional` |
| Исключения | Четыре собственных unchecked-исключения, обработка checked `IOException` |
| Многопоточность | Фоновый поток обновления цен, `Runnable`, `synchronized`, `volatile` |
| File I/O | `java.nio.file.Files` — сохранение и загрузка истории сделок |
| Архитектура | Разделение на слои: модель / ордера / рынок / сервисы / хранение / интерфейс |

## Структура проекта

```
src/
├── model/
│   ├── Stock.java         # Акция (тикер, компания, сектор, изменяемая цена)
│   ├── Position.java      # Позиция в портфеле (количество, средняя цена)
│   ├── Portfolio.java     # Портфель: баланс и позиции
│   ├── Trade.java         # Совершённая сделка (иммутабельная запись)
│   ├── Sector.java        # Enum сектора экономики
│   └── TradeType.java     # Enum типа сделки (покупка / продажа)
│
├── order/
│   ├── Order.java         # Абстрактный ордер (базовый класс)
│   ├── MarketOrder.java   # Рыночный ордер — исполняется немедленно
│   └── LimitOrder.java    # Лимитный ордер — исполняется при достижении цены
│
├── market/
│   ├── Market.java        # Биржа: хранит акции, потокобезопасный доступ к ценам
│   └── PriceUpdater.java  # Фоновый поток, обновляющий цены в реальном времени
│
├── service/
│   ├── TradingService.java    # Логика покупки, продажи, исполнения ордеров
│   └── AnalyticsService.java  # Аналитика портфеля через Stream API
│
├── repository/
│   └── TradeRepository.java   # Сохранение и загрузка истории сделок
│
├── exception/
│   ├── InvalidStockException.java
│   ├── StockNotFoundException.java
│   ├── InvalidOrderException.java
│   └── InsufficientFundsException.java
│
├── ui/
│   └── ConsoleUI.java     # Консольный интерфейс: меню, ввод, запуск потока
│
└── Main.java              # Точка входа: сборка компонентов и запуск
```

Архитектура разделена на слои, каждый со своей ответственностью:
- **model** — данные предметной области (акции, позиции, сделки)
- **order** — типы ордеров с разным поведением исполнения
- **market** — биржа и фоновое обновление котировок
- **service** — бизнес-логика (торговля и аналитика)
- **repository** — сохранение данных
- **ui** — взаимодействие с пользователем

Такое разделение позволяет менять одну часть, не затрагивая другие: например, заменить хранение в файле на базу данных или добавить новый тип ордера без изменения торговой логики.

## Архитектурные решения

**Полиморфизм ордеров.** Абстрактный класс `Order` задаёт контракт через абстрактные методы `canExecute()` и `getExecutionPrice()`. Подклассы реализуют их по-своему: `MarketOrder` исполняется всегда, `LimitOrder` — только когда рыночная цена не превышает лимит. Торговый сервис работает с типом `Order`, не зная конкретного подтипа, — добавление нового вида ордера не потребует изменения сервиса.

**Многопоточность как часть предметной области.** На реальной бирже котировки меняются независимо от действий трейдера. Класс `PriceUpdater` реализует `Runnable` и в фоновом потоке периодически изменяет цены всех акций. Доступ к ценам в `Market` синхронизирован через `synchronized`, а остановка потока реализована через `volatile`-флаг — это гарантирует, что фоновый поток корректно завершится при выходе из приложения.

**Иммутабельность где уместно.** Сделка (`Trade`) неизменяема — это запись о свершившемся факте, её поля объявлены `final`. У акции (`Stock`) изменяемой остаётся только цена, остальные поля неизменны. Это делает поведение объектов предсказуемым.

**Расчёт средней цены позиции.** При докупке акций средняя цена пересчитывается по средневзвешенной формуле, что отражает реальный механизм учёта в брокерских системах.

**Защитное копирование.** Методы, возвращающие коллекции (`getAllStocks`, `getAllPositions`, `getTradeHistory`), отдают копии внутренних структур, защищая состояние объектов от изменения извне.

**Два конструктора сделки.** Один создаёт новую сделку с текущей временной меткой, второй восстанавливает сделку из файла со всеми сохранёнными полями. Основной конструктор делегирует полному через `this(...)`, исключая дублирование валидации.

## Запуск

Требуется Java 14 или выше (используется стрелочный `switch`).

Самый простой способ — открыть проект в IntelliJ IDEA и запустить класс `Main`.

После запуска цены начинают обновляться в фоновом режиме. История сделок сохраняется в файл `trades.txt` при выходе из приложения (пункт меню «Выход»).

## Формат хранения

История сделок сохраняется в `trades.txt`, по одной сделке на строку, поля разделены символом `;`:

```
тикер;тип;количество;цена;время
```

Пример:

```
AAPL;BUY;10;151.23;1782713783185
GAZP;SELL;2;169.85;1782713783248
```
